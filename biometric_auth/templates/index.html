Face recognition with active liveness detection for secure login.

How the pipeline defends the login

01

Password factor (PBKDF2-SHA256)

The face is never the only barrier. A hashed password gate runs first, with 200 000 iterations and per-user salts.

02

Blink detection via EAR

Eye Aspect Ratio tracked frame-by-frame. Printed photos can't blink — the threshold drop is the tell.

03

Three-blink challenge

Server requires three deliberate blinks within the challenge window. Single-frame eye flicker doesn't count — the EAR drop must persist across consecutive frames.

04

512-d ArcFace embedding match

ArcFace ResNet-50 (Deng et al., CVPR 2019) embeddings compared against 3 enrolled samples by cosine distance. Default tolerance 0.40, tunable per experiment.

05

Encrypted templates at rest

Embeddings stored as Fernet-encrypted blobs — an attacker with file access still can't reconstruct identities.

06

Lockout + full audit log

5 failed attempts = 15 min lockout. Every event (login, liveness fail, face mismatch) is logged for review.
